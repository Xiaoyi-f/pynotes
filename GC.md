# python的垃圾回收机制 -> python garbage colection

想象一下你的房间：
你每天都会产生垃圾（不再使用的物品）
如果不清理，房间会越来越乱
最后连走路的地方都没有了

python的垃圾回收就是自动清理"内存垃圾"的机制
当变量不再使用时，python会自动释放他们占用的内存 
没有GC会导致内存泄露问题，内存越用越少，最后程序无法运行而崩溃

GC三机制:
    1.引用计数(主) - 实时回收
    2.标记-清除(辅) - 解决循环引用
    3.分代回收(优化) - 提高效率 

提示: 使用del删除不代表对象就会被回收，删除的只是名字/标签

1.引用计数:
    每个对象都记录着有多少个变量指向我，当指向为零时立即回收
    有点: 实时回收 缺点: 无法处理循环引用 -> 两个对象相互引用
2.标记清除: 
    就像整理房间时，从你站的位置（根对象）出发，给所有能碰到的物品贴上"保留"标签，最后把没贴标签的垃圾直接扔掉
3.分代回收: 
    就像打扫卫生，经常玩的玩具（新对象）每天检查，很少碰的老物件（老对象）偶尔看看，这样既省力又能保持干净
